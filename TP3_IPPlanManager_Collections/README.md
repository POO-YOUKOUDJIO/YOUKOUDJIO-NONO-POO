TP3 : Gestion des réseaux, sous-réseaux et collections d’objets dans IPPlanManager 
	Contexte général du TP 

1. Qu’est-ce qu’une composition en Programmation Orientée Objet ?
La composition est une relation entre classes où un objet est constitué d’autres objets.
2. Pourquoi utilise-t-on ArrayList dans ce TP ?
On utilise ArrayList car c’est une collection dynamique qui permet :d’ajouter ou supprimer facilement des éléments,de stocker un nombre variable d’objets,
parcourir les données avec des boucles.
3. Quelle différence existe entre une variable simple et une collection ?
Variable simple : stocke une seule valeur (ex. une adresse IP).Collection : stocke plusieurs valeurs ou objets du même type (ex. une liste de sous-réseaux).
4. Pourquoi un équipement possède-t-il plusieurs interfaces ?
Un équipement réseau (ex. routeur, switch) doit communiquer sur plusieurs réseaux ou segments.
Chaque interface correspond à une connexion distincte, avec sa propre adresse IP et configuration.
Cela reflète la réalité des équipements professionnels.
5. Pourquoi une infrastructure réseau contient-elle plusieurs sous-réseaux ?
Une infrastructure réseau regroupe différents services ou départements.
Chaque sous-réseau correspond à un besoin spécifique (administration, WiFi, serveurs, etc.).
Cela permet d’organiser, sécuriser et optimiser l’utilisation des adresses IP.
6. Quel est le rôle de la boucle for-each ?
La boucle for-each permet de parcourir facilement tous les éléments d’une collection sans gérer les indices
7. Pourquoi la classe InfrastructureReseau devient-elle importante dans le projet ?
La classe InfrastructureReseau centralise la gestion des sous-réseaux et des équipements.
8. Pourquoi les collections sont-elles indispensables dans les applications professionnelles ?
Les applications professionnelles doivent gérer de grandes quantités de données (utilisateurs, adresses IP, équipements, etc.).

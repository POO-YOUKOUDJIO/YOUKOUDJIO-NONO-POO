TP4 : Calcul automatique des masques, capacités réseau et premières analyses intelligentes dans IPPlan-Manager 

1. Pourquoi a-t-on créé une classe utilitaire ? Une classe utilitaire, comme CalculateurReseau, est créée pour regrouper des traitements qui correspondent à des opérations techniques plutôt qu'à des objets métier réels (comme un équipement ou un utilisateur). Cette approche permet de :  Centraliser les calculs en un seul endroit pour éviter les répétitions de code.  Faciliter la maintenance du projet.  Organiser le code de manière logique en séparant les calculs du reste de l'application.
  2. Quel est le rôle du mot-clé static ? Le mot-clé static indique que la méthode appartient à la classe elle-même plutôt qu'à une instance (objet) spécifique de cette classe.  Conséquence pratique : Il n'est pas nécessaire d'instancier la classe (utiliser new) pour appeler la méthode. On peut l'utiliser directement via le nom de la classe, par exemple : CalculateurReseau.calculerNombreHotes(24). 
 3. Pourquoi les calculs réseau sont-ils importants dans un outil IPAM ? 
Dans un outil de gestion d'adressage IP (IPAM), l'automatisation des calculs est cruciale pour :  Éviter les erreurs humaines lors de la configuration manuelle des masques ou des plages.  Accélérer les déploiements de nouveaux réseaux.  Optimiser les ressources IP en s'assurant que les sous-réseaux sont dimensionnés correctement. 
 4. Quelle est l'utilité du CIDR ? Le CIDR (Classless Inter-Domain Routing) permet de définir un masque de réseau de manière concise (ex: /24) au lieu d'utiliser la notation décimale longue (255.255.255.0). Il offre une plus grande flexibilité pour diviser les réseaux en sous-réseaux de tailles variables, optimisant ainsi l'utilisation de l'espace d'adressage IPv4. 
 5. Pourquoi le nombre d'hôtes dépend-il du masque réseau ? Le masque (ou CIDR) détermine la portion de l'adresse IP réservée à l'identification du réseau et celle réservée aux hôtes.  Plus le CIDR est petit, plus il reste de bits pour les hôtes, augmentant ainsi la capacité du réseau.  La formule utilisée est $2^{(32 - CIDR)} - 2$.
  6. Pourquoi certaines adresses IP sont-elles privées ? Les adresses IP privées (comme les plages 10.x.x.x, 172.16.x.x à 172.31.x.x, et 192.168.x.x) ont été créées pour permettre aux réseaux locaux de fonctionner sans consommer d'adresses IPv4 publiques, qui sont en nombre limité à l'échelle mondiale. Elles ne sont pas routables directement sur l'Internet public. 
 7. Pourquoi la séparation entre logique métier et logique de calcul améliore-t-elle le projet ? 
Cette séparation, aussi appelée découplage, améliore le projet en :  Rendant le code plus lisible et plus facile à tester.  Permettant de modifier les algorithmes de calcul (logique technique) sans impacter la structure des objets réseau (logique métier).
  8. Pourquoi les outils de planification réseau doivent-ils automatiser les calculs ? L'automatisation permet de produire des informations réseau (comme l'adresse de broadcast, la plage utilisable ou la capacité) instantanément et sans risque d'erreur. Cela aide également les techniciens moins expérimentés à gérer des infrastructures complexes sans avoir à effectuer manuellement des conversions binaires ou mathématiques complexes.
